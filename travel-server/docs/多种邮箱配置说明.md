## 📧 邮件发送配置

### 方案1：使用QQ邮箱SMTP（推荐个人项目）

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: your-email@qq.com
    password: your-authorization-code
```

### 方案2：使用163邮箱SMTP

```yaml
spring:
  mail:
    host: smtp.163.com
    port: 465
    username: your-email@163.com
    password: your-authorization-code
    properties:
      mail:
        smtp:
          ssl:
            enable: true
```

### 方案3：使用Gmail SMTP

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: your-app-password
```

### 方案4：使用企业邮箱SMTP（推荐企业项目）

```yaml
spring:
  mail:
    host: smtp.exmail.qq.com  # 腾讯企业邮箱
    port: 587
    username: your-email@company.com
    password: your-password
```

## 🔑 各种邮箱授权码获取方式

### QQ邮箱
1. 登录QQ邮箱网页版
2. 设置 → 账户
3. 开启"POP3/SMTP服务"
4. 获取16位授权码

### 163邮箱
1. 登录163邮箱网页版
2. 设置 → POP3/SMTP/IMAP
3. 开启"SMTP服务"
4. 获取授权码

### Gmail
1. 登录Google账号
2. 安全性 → 两步验证
3. 应用专用密码
4. 生成16位密码

### 企业邮箱
- 联系企业邮箱管理员获取SMTP配置信息

| QQ邮箱 | smtp.qq.com | 465 | 587 | STARTTLS |
| 163邮箱 | smtp.163.com | 465 | 25 | SSL |
| 126邮箱 | smtp.126.com | 465 | 25 | SSL |
| Gmail | smtp.gmail.com | 465 | 587 | STARTTLS |
| Outlook | smtp-mail.outlook.com | 465 | 587 | STARTTLS |

## 🔒 端口和加密配置

### 使用465端口（SSL）
```yaml
spring:
  mail:
    port: 465
    properties:
      mail:
        smtp:
          ssl:
            enable: true
```

### 使用587端口（STARTTLS）
```yaml
spring:
  mail:
    port: 587
    properties:
      mail:
        smtp:
          starttls:
            enable: true
```

## 🎯 用户注册和忘记密码

### 用户侧操作
- 可以使用任意邮箱注册账号
- 可以使用任意邮箱找回密码
- 邮箱格式自动验证

### 系统侧配置
- 只需配置一个SMTP账号用于发送邮件
- 建议使用企业邮箱或专用发送邮箱
- 配置后系统可以发送到任何邮箱

## 💡 推荐配置方案

### 小型项目
- 使用QQ邮箱SMTP
- 配置简单，免费使用
- 适合初期项目

### 中大型项目
- 使用企业邮箱
- 更稳定可靠
- 更专业

### 专业项目
- 使用专业邮件服务（SendGrid、阿里云邮件推送等）
- 高并发支持
- 更详细的统计

## ⚠️ 注意事项

1. **一个SMTP账号发送所有邮件**：系统配置一个SMTP账号，向所有用户邮箱发送
2. **用户邮箱无需特殊配置**：用户可以使用任何邮箱，系统会自动发送
3. **邮件格式统一**：所有邮件使用统一模板和样式
4. **安全建议**：生产环境使用HTTPS安全连接

## 🚀 快速配置

1. 选择一个邮箱供应商（推荐QQ邮箱或企业邮箱）
2. 获取SMTP配置信息
3. 在`application.yml`中配置
4. 重启服务测试发送

## 📞 技术支持

如有问题，请检查：
1. SMTP配置是否正确
2. 授权码是否有效
3. 端口是否可用
4. 防火墙设置
