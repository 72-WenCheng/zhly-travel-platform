# 邮件配置说明

## 配置真实邮箱服务

### 1. QQ邮箱配置（推荐）

#### 步骤：
1. 登录QQ邮箱网页版
2. 点击"设置" -> "账户"
3. 开启"POP3/SMTP服务"或"IMAP/SMTP服务"
4. 获取授权码（不是QQ密码！）

#### 配置示例：
```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: your-qq-email@qq.com  # 你的QQ邮箱
    password: your-authorization-code  # 授权码，不是QQ密码
```

### 2. 网易邮箱配置

#### 163邮箱：
```yaml
spring:
  mail:
    host: smtp.163.com
    port: 465
    username: your-email@163.com
    password: your-authorization-code
```

#### 126邮箱：
```yaml
spring:
  mail:
    host: smtp.126.com
    port: 465
    username: your-email@126.com
    password: your-authorization-code
```

### 3. Gmail配置

```yaml
spring:
  mail:
    host: smtp.gmail.com
    port: 587
    username: your-email@gmail.com
    password: your-app-password  # 需要使用应用专用密码
```

### 4. 企业邮箱配置

#### 腾讯企业邮箱：
```yaml
spring:
  mail:
    host: smtp.exmail.qq.com
    port: 465
    username: your-email@yourcompany.com
    password: your-password
```

## 使用环境变量配置（推荐）

在IDE运行配置或系统环境变量中设置：

```bash
MAIL_HOST=smtp.qq.com
MAIL_PORT=587
MAIL_USERNAME=your-email@qq.com
MAIL_PASSWORD=your-authorization-code
```

## 开发环境 - 不配置真实邮箱

如果不配置邮箱信息，系统会自动使用模拟发送模式：
- 验证码会在后端控制台打印
- 不会实际发送邮件
- 方便开发和测试

## 注意事项

1. **授权码不是邮箱密码**：QQ邮箱、163邮箱等需要使用授权码，而不是登录密码
2. **端口选择**：
   - 587端口：使用STARTTLS加密
   - 465端口：使用SSL加密
3. **防火墙**：确保服务器可以访问邮件服务器的端口
4. **发送频率限制**：避免频繁发送邮件，可能被邮件服务商限制

## 测试邮件功能

配置完成后，重启后端服务，测试发送验证码功能：
1. 访问忘记密码页面
2. 输入任意邮箱地址
3. 点击"获取验证码"
4. 检查邮箱收件箱或后端控制台输出
