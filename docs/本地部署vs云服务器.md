# 本地部署 vs 云服务器部署

## 🖥️ 本地部署（你的电脑）

### 你的理解 ✅

> "本地部署就是自己的电脑要一直开着，然后别人要访问你的项目地址还需要搭建局域网吗"

**基本正确！** 但需要区分两种情况：

### 情况1：局域网访问（需要搭建局域网）✅

```
你的电脑（192.168.1.100）
    ↓
  路由器
    ↓
同一WiFi下的设备
├── 室友的电脑（192.168.1.101）
├── 你的手机（192.168.1.102）
└── 其他设备

访问方式：
http://192.168.1.100:3000  （前端）
http://192.168.1.100:8070   （后端）

特点：
✅ 不需要额外配置
✅ 同一网络下都能访问
❌ 外网的人访问不了
❌ 你的电脑必须开着
```

**实际场景：**
- 你在宿舍，室友想访问你的项目
- 公司内部，同事想访问你的项目
- 家庭网络，家人想访问

**需要做什么：**
1. 你的电脑和访问者的设备连同一个WiFi/网络
2. 你的电脑运行项目（后端+前端）
3. 访问者输入你的内网IP（如：192.168.1.100）

**不需要做什么：**
- ❌ 不需要买服务器
- ❌ 不需要买域名
- ❌ 不需要特殊配置（通常）

---

### 情况2：外网访问（需要特殊配置）⚠️

```
你的电脑（192.168.1.100）
    ↓
  路由器（没有公网IP）
    ↓
  互联网
    ↓
  外网用户想访问

问题：
❌ 外网用户无法直接访问你的内网IP
❌ 需要内网穿透或端口转发
```

**解决方案：**

#### 方案A：内网穿透工具（简单）

使用工具如：**ngrok**、**frp**、**花生壳**

```
你的电脑 → 内网穿透工具 → 公网地址 → 外网用户

例子：
1. 安装 ngrok
2. 运行：ngrok http 3000
3. 获得公网地址：https://abc123.ngrok.io
4. 外网用户访问这个地址
```

**优点：**
- ✅ 简单快速
- ✅ 免费（有限制）
- ✅ 不需要配置路由器

**缺点：**
- ❌ 免费版有流量限制
- ❌ 地址会变化（免费版）
- ❌ 速度可能较慢

#### 方案B：路由器端口转发（复杂）

```
你的电脑（192.168.1.100:8070）
    ↓
  路由器（配置端口转发）
    ↓
  公网IP（123.456.789.012:8070）
    ↓
  外网用户访问
```

**需要条件：**
- ✅ 路由器有公网IP（现在很多都没有）
- ✅ 会配置路由器
- ✅ 运营商允许（有些运营商禁止）

**缺点：**
- ❌ 配置复杂
- ❌ 很多家庭网络没有公网IP
- ❌ 安全性需要额外配置

---

## ☁️ 云服务器部署（推荐）

### 你的理解 ✅

> "云服务器就是人家的云端服务器一直开着的，部署了大家都可以访问"

**完全正确！**

### 详细说明

```
云服务器（24小时运行）
├── 公网IP：123.456.789.012
├── 域名：www.yourwebsite.com（可选）
├── 你的项目
└── 全世界都能访问

访问方式：
http://123.456.789.012        （直接用IP）
http://www.yourwebsite.com    （用域名，更好记）

特点：
✅ 24小时运行
✅ 全世界都能访问
✅ 有公网IP
✅ 稳定可靠
✅ 可以买域名绑定
```

**实际场景：**
- 正式上线的项目
- 需要让全世界访问
- 需要稳定运行

**需要做什么：**
1. 购买云服务器（腾讯云/阿里云等）
2. 部署你的项目
3. 配置域名（可选）
4. 完成！

**优点：**
- ✅ 专业稳定
- ✅ 全世界可访问
- ✅ 有技术支持
- ✅ 可以扩展（升级配置）

**缺点：**
- ❌ 需要花钱（几十到几百/月）
- ❌ 需要学习部署

## 🎯 实际使用建议

### 场景1：本地开发
```
你的电脑
├── 运行后端：localhost:8070
├── 运行前端：localhost:3000
└── 只有你能访问

用途：开发、调试
```

### 场景2：内网测试
```
你的电脑（192.168.1.100）
    ↓
同一WiFi
    ↓
室友访问：http://192.168.1.100:3000

用途：让室友/同事测试
需要：你的电脑开着，同一网络
```

### 场景3：外网临时访问
```
你的电脑
    ↓
内网穿透工具（ngrok）
    ↓
公网地址：https://abc123.ngrok.io
    ↓
外网用户访问

用途：临时演示、测试
需要：你的电脑开着，安装内网穿透工具
```

### 场景4：正式上线（推荐）
```
云服务器
    ↓
公网IP/域名
    ↓
全世界访问

用途：正式项目
需要：购买云服务器，部署项目
```

## 💡 常见问题

### Q1: 本地部署可以让外网访问吗？

**可以，但需要：**
1. 内网穿透工具（简单，推荐）
2. 或路由器端口转发（复杂）

**但建议：**
- 临时测试 → 用内网穿透
- 正式上线 → 用云服务器

### Q2: 云服务器需要我的电脑开着吗？

**不需要！**
- 云服务器是独立的电脑
- 24小时运行
- 你的电脑可以关机

### Q3: 本地部署和云服务器哪个好？

**看用途：**
- **开发测试** → 本地部署
- **正式上线** → 云服务器
- **临时演示** → 本地+内网穿透
- **长期运行** → 云服务器

### Q4: 内网穿透工具推荐？

**免费工具：**
- ngrok（最流行）
- frp（开源）
- 花生壳（国内）

**使用方法（ngrok）：**
```bash
# 1. 下载 ngrok
# 2. 注册账号，获取 token
# 3. 运行
ngrok http 3000

# 4. 获得公网地址
# Forwarding: https://abc123.ngrok.io -> http://localhost:3000
```

### 我想让外网临时访问

- 获得公网地址分享

### 我想正式上线项目
**选择：云服务器**
- 购买云服务器
- 部署项目

## 📝 总结

### 你的理解 ✅

1. ✅ **本地部署需要电脑一直开着** - 正确
2. ✅ **局域网访问需要搭建局域网** - 正确（同一网络即可）
3. ✅ **云服务器一直开着，大家都能访问** - 完全正确

### 补充知识

1. **局域网访问** = 同一网络下的设备访问（简单）
2. **外网访问本地** = 需要内网穿透工具（复杂，不推荐长期用）
3. **云服务器** = 专业、稳定、推荐正式项目使用
